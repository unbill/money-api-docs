    Resource URL Patterns

    /merchant

    Welcome! The Q2 Money API allows developers to integrate with Request Money, a white-label solution that provides small business's a digital way to request money from their customers.

    Here you will find information on how to integrate with and use the Q2 Money API. We've tried to make this documentation user-friendly and example-filled, but please drop us a line with any questions.

    The Q2 Money API uses standard HTTP verbs to communicate and HTTP response codes to indicate status and errors. All responses come in standard JSON. The Q2 Money API is served over HTTPS to ensure data privacy, HTTP is not supported.

    Sandbox Limitations

    • You cannot move real money on Sandbox.

    Gaining Access

    To gain access to the Q2 Money API, please contact us. After our review process is complete, you'll be provided with an API key which you can use for your requests.

    Authentication

    Example Request

      curl "https://api.q2money.com/v1/api/merchant/sync"
    -H "Authorization: Bearer dc220490-e6ee-11e5-8a94-e7385a8d929e"

    Authenticate your account when using the API by including your secret API key in the request. Your API keys carry many privileges, so be sure to keep them secret! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

    Authentication to the API is performed via bearer tokens. Pass your API Key as a bearer token in the Authorization header.

    Errors

    The Q2 Money API uses HTTP response status codes to indicate the success or failure of your API requests. If your request fails, Q2 Money API returns an error using the appropriate status code.

    In general, there are three status code ranges you can expect:

    • 2xx success status codes confirm that your request worked as expected
    • 4xx error status codes indicate an error because of the information provided (e.g., a required parameter was omitted)
    • 5xx error status codes are rare and indicate an error with Q2 Money API's servers

    White-label

    The Q2 Money White-label application allows small business's to request money digitally from their customers.

    When a payment request is sent to a customer, the customer is provided with a secure link via email to pay the small business.

    To integrate with the white-label, you must provide the Q2 Money API details about the small business via the Merchant API.

    Creates or updates a merchant and returns a ssoToken to be used with the white-label application.

    HTTP Request

    POST https://api.q2money.com/v1/api/merchant/sync

    Arguments

    Parameter Description
    identifier A unique identifier that represents a merchant in your system.
    meta Additional information about the merchant in your system. This information will be used for identity verification by our payment processor in order to accept payments.
    meta.bankAccounts A list of bank accounts that the merchant will select from as their deposit account.

    Returns

    Returns an object with a ssoToken property that can be used with the white-label application.
